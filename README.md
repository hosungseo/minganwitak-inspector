# 국가사무 민간위탁 워크스페이스

> 민간위탁 **공개데이터 구조**와 **실태점검 운영 화면**을 함께 다루는 작업용 저장소입니다.

이 저장소는 두 축을 함께 담고 있습니다.

1. **민간위탁 공개데이터 구조화**  
   - 행정기관 홈페이지에 공개되는 민간위탁사무 정보를 수집·표준화·통합하기 위한 스키마와 문서
2. **민간위탁 실태점검 인터랙티브 데모**  
   - JSX v5를 기반으로 한 React/Vite 프론트엔드 데모

**Public demo:** <https://hosungseo.github.io/minganwitak-inspector/>

## 왜 이 저장소가 필요한가

`행정권한의 위임 및 위탁에 관한 규정` 제13조 제3항(2025.3.4. 신설, 2026.3.1. 시행)에 따라, 위탁기관은 민간수탁기관과의 계약 체결 시 소관 민간위탁사무의 내용·근거·수탁기관·위탁기간 등 계약 관련 사항을 인터넷 홈페이지에 공개해야 합니다.

이 저장소는 그 제도 변화에 대응해,
- 공개 대상 데이터의 **공통 구조를 잡고**
- 실태점검과 공개관리를 실제로 어떻게 **운영 화면으로 풀 수 있는지**
함께 실험하기 위한 워크스페이스입니다.

## 포함 내용

### 1) 데이터/문서 레이어

```text
data/
  기관코드/
    YYYY-MM.json       # 월별 수집 데이터
scrapers/              # 기관별 수집 스크립트
schema/                # 데이터 스키마 정의
docs/                  # 활용 가이드
```

### 2) 프론트엔드 데모 레이어

```text
src/App.jsx            # 민간위탁 실태점검 v5 기반 메인 앱
src/main.jsx           # React 엔트리
source/민간위탁_실태점검_v5.jsx   # 메일 첨부 원본 보관
.github/workflows/deploy-pages.yml # GitHub Pages 배포 워크플로
```

## 데모 앱 개요

앱 제목은 **국가사무 민간위탁 실태점검 시스템**이며, 다음 흐름을 한 화면에서 다룹니다.

- 위탁사무 목록 관리
- 연도별 이월
- 제출 / 보완요청 / 확정 상태 관리
- 공개 승인 및 JSON 내보내기
- 대시보드 / 통계 / 공개관리 화면

프론트는 단순한 관리툴 스킨이 아니라, **운영형 제품 UI**로 다듬어 두었습니다.
디자인 방향은 다음 원칙을 따릅니다.

- 카드 남발보다 **조용한 정보 위계**
- 첫 화면에서 바로 업무 맥락이 보이는 **운영형 hero / workspace intro**
- 강한 한 가지 액센트와 절제된 중립색
- 테이블·필터·상태 관리가 빠르게 읽히는 구조

## 데이터 스키마 (초안)

| 필드 | 설명 | 예시 |
|------|------|------|
| `agency_code` | 위탁기관 코드 | `1741000` |
| `agency_name` | 위탁기관명 | `행정안전부` |
| `task_name` | 민간위탁사무 내용 | `정부전산센터 운영` |
| `legal_basis` | 법적 근거 | `전자정부법 제45조` |
| `contractor` | 민간수탁기관 | `○○공단` |
| `contract_start` | 위탁 시작일 | `2026-01-01` |
| `contract_end` | 위탁 종료일 | `2028-12-31` |
| `fee_or_cost` | 위탁 수수료/비용 | `연 5억원` |
| `source_url` | 원본 공개 페이지 | `https://...` |
| `collected_at` | 수집 일시 | `2026-03-25T14:00:00Z` |

## 공개 예외

제13조 제4항에 따라 **국가안전보장·외교·국방** 등 성질상 공개가 적절하지 않은 사무는 제외될 수 있습니다.
